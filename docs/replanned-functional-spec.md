# Axiss Nav 重新规划功能文档

本文档综合当前已有功能和后续改良建议，作为下一阶段产品与技术实现的目标规划。内容只覆盖功能和业务边界，不包含前端排版设计。

## 1. 产品定位

Axiss Nav 是一个单管理员维护的个人智能导航系统，用于长期收藏、整理、分析和维护常用网站链接。

系统核心目标不是简单保存 URL，而是让链接库具备持续自整理能力：

- 快速保存链接。
- 自动理解链接内容。
- 自动生成标题、描述、标签和分类建议。
- 发现重复、失效、未整理和低质量链接。
- 支持批量整理和备份迁移。

系统不做多用户平台，不提供注册、用户管理、团队协作和权限分层。

## 2. 管理员认证

系统只保留一个预设管理员账户。

管理员认证信息由环境变量或部署配置提供：

- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`
- `JWT_SECRET`

登录成功后返回 JWT，前端保存到 `localStorage`，后续管理接口通过 `Authorization: Bearer <token>` 校验。

不提供：

- 用户注册
- 普通用户登录
- 用户表
- 用户角色
- 管理员初始化页面
- 密码找回
- 用户资料管理

## 3. 访问权限

公开访问者可以：

- 查看链接列表。
- 搜索链接。
- 按标签和分类筛选链接。
- 使用排序和快捷筛选。
- 打开链接。
- 触发点击统计。
- 查看热门标签。
- 使用随机访问。
- 查看推荐或发现入口。

管理员登录后可以：

- 添加链接。
- 编辑链接。
- 删除链接。
- 重新分析链接。
- 批量分析链接。
- 批量管理标签和分类。
- 导入链接。
- 导出链接。
- 手动触发备份。
- 查看链接健康状态和整理建议。

## 4. 链接管理

链接是系统唯一核心业务对象。

每条链接包含：

- 标题
- URL
- 描述
- 图标地址
- 排序值
- 是否启用
- 点击次数
- 标签
- 主分类
- 分类颜色
- 分析状态
- 最近分析时间
- 最近健康检查时间
- 可访问状态
- 创建时间
- 更新时间

链接 URL 全局唯一。

删除链接采用软删除，不直接从数据库物理删除。软删除链接可在导入或重复检测时被识别，并支持恢复。

## 5. 添加链接

管理员可以手动添加链接，也可以通过智能分析 Agent 辅助添加。

添加流程：

1. 输入 URL，或从剪贴板自动识别 URL。
2. 校验 URL 是否为 `http://` 或 `https://`。
3. 检查是否存在重复链接。
4. 获取网页标题、描述、favicon 和正文摘要。
5. 调用智能分析 Agent 生成建议。
6. 管理员确认或修改后保存。

添加时允许管理员手动覆盖：

- 标题
- 描述
- 图标
- 标签
- 分类
- 颜色

如果分析失败，系统仍允许以基础信息保存链接。

## 6. 智能分析 Agent

智能分析 Agent 是系统重点能力，用于让链接库可自动整理、可维护、可持续优化。

Agent 能力包括：

- 网页信息提取：读取标题、meta 描述、关键词、favicon 和正文摘要。
- 内容理解：识别网站用途、内容领域、工具类型和适用场景。
- 标题优化：去掉冗余站点后缀，生成适合收藏展示的标题。
- 描述生成：生成简短、准确、可读的中文描述。
- 标签生成：生成 1 到 5 个稳定标签。
- 分类推荐：从已有分类中优先选择主分类，必要时建议新分类。
- 重复识别：识别完全重复 URL、同域名相似链接、跳转链接和同类工具链接。
- 失效检测：检测链接无法访问、跳转异常、证书异常和 favicon 失效。
- 批量整理：对已有链接批量补全标题、描述、图标、标签和分类。
- 维护建议：发现未分类、标签混乱、描述为空、长期未访问和可能失效链接。

Agent 输出原则：

- 只提供建议，不默认覆盖管理员手动编辑内容。
- 优先复用已有标签和分类。
- 不编造网页不存在的能力。
- 分析失败时回退到基础网页信息。
- 对低置信度建议进行标记。

## 7. 重新分析

重新分析用于更新已有链接的信息。

管理员可以对单条链接执行重新分析，也可以批量选择链接重新分析。

重新分析内容包括：

- 标题建议
- 描述建议
- 图标更新
- 标签建议
- 分类建议
- 重复风险
- 可访问状态

重新分析结果进入待确认状态。

管理员可以：

- 应用全部建议。
- 逐项应用建议。
- 忽略建议。
- 保留手动编辑内容。

## 8. 搜索、筛选与排序

链接列表支持分页加载。

搜索范围：

- 标题
- URL
- 描述
- 标签
- 分类

筛选能力：

- 按标签筛选。
- 按分类筛选。
- 仅看常用链接。
- 仅看最近添加链接。
- 仅看未分类链接。
- 仅看未分析链接。
- 仅看失效链接。
- 仅看疑似重复链接。

排序能力：

- 最新添加
- 最早添加
- 点击最多
- 标题 A-Z
- 最近分析时间
- 最近健康检查时间

## 9. 标签体系

标签应从 JSON 字符串升级为结构化模型。

标签 `Tag` 包含：

- id
- name
- slug
- icon
- color
- 使用次数
- 创建时间
- 更新时间

链接和标签通过关联表维护多对多关系。

标签管理能力：

- 新建标签。
- 改名标签。
- 删除标签。
- 合并标签。
- 批量替换标签。
- 查看标签使用次数。
- 查看未使用标签。

Agent 可参与标签治理：

- 发现近义标签。
- 建议合并标签。
- 建议删除低价值标签。
- 为未打标签链接补充标签。

## 10. 分类体系

分类用于表达链接的主领域，应保持少量、稳定、互斥。

每条链接最多有一个主分类，可以有多个标签。

分类管理能力：

- 新建分类。
- 修改分类名称。
- 修改分类颜色。
- 删除分类。
- 批量移动链接分类。
- 查看分类下链接数量。

Agent 生成分类时应优先从已有分类中选择，避免不断创建相似分类。

## 11. 点击统计与发现入口

系统记录每条链接的点击次数。

点击统计用于：

- 展示使用频率。
- 支持“点击最多”排序。
- 辅助发现低访问链接。
- 辅助推荐或发现入口。

推荐入口应更准确命名为“发现链接”或“轮换入口”。

发现算法可以综合：

- 点击次数。
- 添加时间。
- 最近访问情况。
- 是否长期未访问。
- 随机因子。
- 链接健康状态。

系统也保留随机访问能力，从活跃链接中随机打开一个链接。

## 12. 链接健康检查

系统应支持链接健康检查。

检查内容：

- URL 是否可访问。
- 是否发生异常跳转。
- 是否返回 404、500 等错误。
- HTTPS 证书是否异常。
- favicon 是否失效。

健康状态：

- 正常
- 无法访问
- 跳转异常
- 服务器错误
- 检查失败
- 未检查

健康检查可手动触发，也可定期执行。

## 13. 批量管理

管理员可以对链接执行批量操作。

批量操作包括：

- 批量删除。
- 批量恢复。
- 批量修改分类。
- 批量添加标签。
- 批量移除标签。
- 批量重新分析。
- 批量健康检查。
- 批量导出。

批量操作应提供确认步骤，避免误操作。

## 14. 导入导出

系统支持 JSON 和 Markdown 导入导出。

导出内容包括：

- 链接基础信息。
- 标签。
- 分类。
- 点击次数。
- 分析状态。
- 健康检查状态。
- 创建和更新时间。

导入流程：

1. 上传 JSON 或 Markdown 文件。
2. 解析链接数据。
3. 校验标题和 URL。
4. 检测重复 URL。
5. 检测软删除链接。
6. 展示导入预览。
7. 管理员确认后批量导入。

重复处理策略：

- 已存在活跃链接：跳过或更新。
- 已存在软删除链接：恢复或重新创建。
- 格式错误链接：列入错误清单。

## 15. 备份功能

系统保留本地备份、GitHub 自动备份和 Vercel 定时备份。

备份能力：

- 手动触发备份。
- 定时自动备份。
- 导出 JSON 备份。
- 导出 Markdown 备份。
- 推送到 GitHub 仓库。
- 展示最近一次备份状态。

备份配置：

- `GITHUB_BACKUP_REPO`
- `GITHUB_TOKEN`
- `BACKUP_BRANCH`
- `BACKUP_DIR`

备份失败时应记录错误信息，方便管理员排查。

## 17. 实施优先级

第一阶段：简化认证与数据基础

- 移除注册能力。
- 移除初始化管理员流程。
- 移除用户表。
- 改为配置式单管理员登录。
- 保留现有链接增删改查、搜索、筛选和导入导出。

第二阶段：强化智能分析 Agent

- 统一添加分析和重新分析逻辑。
- 增加分析状态字段。
- 增加待确认分析建议。
- 支持单条重新分析。
- 支持批量分析。

第三阶段：结构化标签和分类

- 增加 `Tag`、`LinkTag`、`Category` 模型。
- 迁移现有 JSON 标签。
- 增加标签合并、改名和批量替换。
- 增加分类管理。

第四阶段：健康检查与维护建议

- 增加链接健康检查。
- 增加重复链接识别。
- 增加未分类、未分析、低质量描述筛选。
- 增加 Agent 整理建议。

第五阶段：批量管理和备份体验

- 增加批量操作。
- 增加导入预览。
- 增加手动备份入口。
- 增加最近备份状态展示。

## 18. 非目标功能

当前规划不包含：

- 多用户注册登录。
- 团队协作。
- 权限分级。
- 公开投稿。
- 评论系统。
- 收藏夹社交分享。
- 浏览器插件。
- 移动端原生 App。
